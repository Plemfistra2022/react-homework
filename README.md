# Урок: Маршрутизация в React с использованием React Router v7

В этом уроке мы внедрили навигацию в приложение, научились работать с лейаутами и дочерними маршрутами.

## 📦 Новые зависимости
Для работы с путями была установлена библиотека:
* `react-router-dom` (версия 7+)

---

## 🛠 Ключевые изменения и концепции

### 1. Конфигурация Router (`main.tsx`)
Мы перешли на декларативный способ описания маршрутов с помощью `createBrowserRouter`. Это позволяет четко видеть структуру приложения в одном месте.

```typescript
const router = createBrowserRouter([
    {
        path: '/', 
        element: <MainLayout/>, // Общая оболочка для всех страниц
        children: [
            { path: '', element: <HomePage/> }, // Главная страница (/)
            { path: 'a', element: <PageA/> },   // Страница A (/a)
            { path: 'b', element: <PageB/> },   // Страница B (/b)
        ]
    },
]);
```

### 2. Layout-ориентированный подход
Мы создали `MainLayout`, который служит каркасом для нашего приложения.
* **`<Menu/>`**: Отображается всегда, на любой странице.
* **`<Outlet/>`**: Специальный компонент-заглушка, на место которого React Router подставляет контент текущего дочернего маршрута (`HomePage`, `PageA` или `PageB`).

### 3. Навигация без перезагрузки (`Link`)
Вместо стандартных тегов `<a href="...">`, которые перезагружают страницу, мы используем компонент `<Link to={'...'}>`. Это позволяет переключать страницы мгновенно, сохраняя состояние приложения (SPA — Single Page Application).

---

## 📂 Структура проекта

* **`layouts/`**: Содержит компоненты-оболочки (например, `MainLayout`).
* **`pages/`**: Содержит полноценные страницы приложения.
* **`components/menu/`**: Навигационная панель.

---

## 🚀 Как это работает
1. Пользователь заходит на сайт. `RouterProvider` анализирует URL.
2. Отрисовывается `MainLayout`.
3. Внутри `MainLayout`, на месте `<Outlet/>`, отрисовывается компонент, соответствующий пути (например, если путь `/a`, то отрисуется `PageA`).

---

## 📝 Запуск
1. Установите зависимости: `npm install`.
2. Запустите проект: `npm run dev`.
3. Попробуйте покликать по ссылкам в меню — контент под чертой (`<hr/>`) будет меняться без перезагрузки всей страницы.
