В этой итерации мы вынесли логику запросов в отдельный слой сервисов (API Layer) и перешли на использование асинхронных функций.

🛠 Что изменилось
Separation of Concerns (Разделение ответственности): Логика fetch теперь живет в api.service.ts, а не внутри компонента.

Async/Await: Вместо цепочек .then(), внутри useEffect используется асинхронная функция fatchData.

Архитектура сервиса: Созданы переиспользуемые функции для получения списка всех пользователей и одного конкретного пользователя.

📂 Структура сервиса (api.service.ts)
Вынос запросов в отдельный файл позволяет легко менять базовый URL или библиотеку для запросов (например, на Axios) в одном месте.

TypeScript
export const getUsers = async (): Promise<IUser[]> => {
  return await fetch('https://jsonplaceholder.typicode.com/users')
  .then(res => res.json())
}

💻 Обновленный компонент UsersComponent
Теперь компонент стал более декларативным. Он просто "просит" сервис предоставить данные.

Ключевые моменты:
Внутренняя асинхронная функция: Поскольку коллбэк в useEffect не может быть асинхронным напрямую, мы создаем async fatchData внутри.

Именованный экспорт: Перешли от export default к именованному экспорту компонентов для лучшей поддержки IDE и рефакторинга.
