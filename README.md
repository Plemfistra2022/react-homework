Урок: Router demo

🏗 Что было сделано
1. Маршрутизация (React Router v7)
   Централизованный роутинг: Все пути приложения теперь описаны в одном конфиге routes.tsx.

Layout-паттерн: Использование общего компонента Layout с Outlet позволяет сохранять общие элементы (меню, футер) при переключении страниц.

Индексация: Настроили index: true для отображения главной страницы по умолчанию.

2. Организация страниц и компонентов
   Разделили проект на Pages (целые экраны) и Components (переиспользуемые части, такие как Menu).

Созданы отдельные страницы для пользователей (UsersPage) и постов (PostsPage).

3. Слой API и констант
   Конфигурация URL: Все эндпоинты вынесены в объект urls. Больше никаких "хардкод" строк в компонентах!

Динамические ссылки: Реализовали функции-помощники для генерации путей с параметрами (например, получение пользователя по ID).

📂 Итоговая структура
src/router/ — конфигурация путей.

src/pages/ — компоненты страниц.

src/layouts/ — общие шаблоны.

src/constants/ — все внешние ссылки и настройки API.

src/components/ — мелкие UI-элементы.
